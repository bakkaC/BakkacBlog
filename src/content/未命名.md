
项目功能清单：

11. F11 Provider体系：内置Provider + 自定义Provider，增删改查与模型列表维护。

13. F13 模型能力管理：模型能力测试（vision/tool_use）、默认模型、收藏模型。
14. F14 Chatbox AI账号体系：登录/手动License激活、实例绑定、失效回收。
15. F15 Copilots：本地Copilot创建编辑收藏 + 远端精选Copilot。
16. F16 Web搜索工具：Chatbox Search/Bing/Tavily，可注入模型工具调用。
17. F17 文件工具调用：对附件文件 read_file/grep_file 工具。
18. F18 知识库RAG（桌面）：知识库CRUD、文件入库、向量检索、失败重试/暂停恢复。
19. F19 知识库工具集：query_knowledge_base/get_files_meta/read_file_chunks/list_files。
20. F20 MCP（桌面）：内置MCP + 自定义MCP（HTTP/STDIO），工具注入模型。
21. F21 MCP安装体验：MCP注册表、JSON导入、深链安装（chatbox://mcp/install?...）。
22. F22 OCR兜底：非视觉模型自动OCR图片再入模。
23. F23 自动标题：会话名/线程名自动生成。
24. F24 渲染能力：Markdown/LaTeX/Mermaid/Artifact预览/代码块折叠。
25. F25 多语言：多语种i18n。
26. F26 存储与迁移：Desktop(File+IndexedDB)/Web(IndexedDB)/Mobile(SQLite)迁移体系。
27. F27 数据维护：配置自动备份、会话恢复、设置与数据导入导出。
28. F28 桌面系统集成：托盘、全局快捷键、窗口控制、系统主题监听、深链、代理、开机启动。
29. F29 自动更新：多源检查、beta通道、下载后安装。

# Chatbox 极简重构方案

## 摘要

本轮按你的最新要求收敛为：

1. 去掉联网搜索功能（原 F16 全删）。
2. 去掉旧数据策略（不迁移、不清库、不兼容转换）。
3. Provider 简化到“只填一个 API Key 即可用”，底层固定为单一 OpenAI 兼容接入。

保留目标能力变为：

1. F04 线程能力（新线程/回退线程/线程历史）。
2. F06 消息内容模型（精简版）。
3. F09 会话设置（仅名称 + System Prompt）。
4. F14 基础登录与登出（本地凭据管理）。
5. F17 文件工具（仅 read_file）。
6. F27 数据导入/导出。

## 实施范围（最终）

1. 删除功能：F03/F05/F11/F12/F13/F15/F16/F18/F19/F20/F21/F22/F25/F28/F29/F30。
2. 简化功能：  
    F14 仅登录/登出。  
    F26 保留存储体系，移除迁移体系与旧数据处理流程。  
    F07 仅保留文本文件 + 图片附件。
3. 必保功能：F04/F06/F09/F14/F17/F27。

## 关键实现设计

1. 平台收敛为仅桌面端，移除 Web/Mobile 分支逻辑和 feature flag 分叉。
2. Provider 收敛为单 provider：  
    只保留一个设置项 apiKey。  
    apiHost 固定为内置常量 SINGLE_PROVIDER_API_HOST（OpenAI 兼容网关）。  
    modelId 固定默认值（例如 gpt-4o-mini），不暴露 provider/model 管理 UI。
3. 删除联网搜索：移除 web-search 包、设置页入口、InputBox 的 web-browsing 开关、模型调用中的 web 工具注入与降级逻辑。
4. 工具系统仅保留 read_file：删除 grep_file、MCP 工具、知识库工具、联网搜索工具。
5. 会话仅保留 chat 类型：删除图片会话路由入口、图片会话生成逻辑、相关按钮与设置项。
6. 消息模型精简：保留 text/image/tool-call/info；删除与 fork、知识库、MCP、联网搜索绑定字段和状态分支。
7. 线程能力保留：保留新线程/回退线程/历史抽屉；删除 fork 数据结构与所有 fork 操作函数和 UI。
8. 会话设置精简：仅保留“会话名 + System Prompt”；删除会话级模型参数、头像、温度/TopP/maxContext 面板。
9. 登录体系（F14）：保留最小登录态持久化与登出清理；移除 license 激活、套餐、实例校验、相关远端接口。

10. 存储与启动（F26）：删除 migration 调用与版本迁移代码；应用启动直接读取当前存储，解析失败回默认值，不做旧结构转换。
11. 数据维护（F27）：保留导入/导出；删除恢复扫描、自动备份策略、旧版本恢复逻辑。
12. 主进程精简：删除托盘、全局快捷键、深链、自动更新、代理、埋点与 Sentry 初始化；保留窗口基础 IPC、存储 IPC、本地文件解析 IPC。

## 对外接口与类型变更

1. Settings：删除 providers/customProviders/favoritedModels/default-models/webSearch/mcp/knowledgeBase/license* 等字段，新增精简结构：  
    provider.apiKey、auth.session、chatDefaults.systemPrompt。
2. SessionType：仅 chat。
3. SessionSettings：只保留与会话本身相关的最小字段（名称、system prompt 映射）。
4. 模型工厂：getModel 不再按 provider 分发，改为单实现构造。
5. 删除 IPC/路由：/settings/web-search、MCP、知识库、provider 管理、deep link 安装导入通道。

## 测试与验收

1. 启动与主流程：应用可启动，能发送消息并流式返回。
2. 线程：新线程、回退线程、线程历史切换可用。
3. 会话设置：可编辑名称与 System Prompt 并持久化。
4. 附件：文本文件与图片可上传；read_file 可读取文本文件内容。
5. 登录：登录成功后持久化；登出后凭据清空。
6. 数据导入导出：导出后可在新环境导入恢复。
7. 删除项回归：无 web-search、无 MCP、无知识库、无 provider 切换入口、无自动更新入口。
8. 构建检查：npm run check、npm run test 通过；删除模块后无死引用。

## 假设与默认值

1. 单 API Key 接入采用“固定 OpenAI 兼容网关 + 固定默认模型”。
2. 网关地址使用内置常量 SINGLE_PROVIDER_API_HOST（构建时可改常量，不对用户暴露）。
3. 不提供任何旧数据迁移、清理、兼容转换逻辑。
